# 实验步骤

## 1. 实验准备
- 安装 Kali Linux
- 配置桥接网络
- 配置 DNS
- 确保实验环境处于可控局域网中

## 2. 主机扫描
使用扫描方法查看同一局域网中的其他主机，确认目标设备。

## 3. ARP 欺骗实验
使用 arpspoof 工具发起 ARP 欺骗攻击，模拟攻击者向目标主机与网关伪造 ARP 响应。

## 4. 攻击验证
- 查看目标机 ARP 缓存表
- 验证目标主机是否将攻击者误认为网关
- 观察是否成功实现流量中转

## 5. 数据监听
在攻击者主机上监听经过的数据包，分析攻击效果。

## 6. 实验结论
ARP 欺骗攻击利用了 ARP 协议缺乏身份验证机制的弱点，可导致流量劫持、中间人攻击或断网效果。
